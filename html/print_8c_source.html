1 // Estrutura elementar de um ficheiro de implementação
2 // ===================================================
3 //
4 // Este é um dos ficheiros de implementação associados ao ficheiro de cabeçalho
5 // [`array_utils.h`](array_utils.html). Neste caso optou-se por manter um único ficheiro de
6 // cabeçalho e por dividir a implementação por vários ficheiros de
7 // implementação. Dessa forma:
8 //
9 // - Simplifica-se a vida de quem pretende utilizar os utilitários, pois terá de
10 // incluir apenas um ficheiro de cabeçalho. Claro está que esta vantagem se
11 // pode transformar numa desvantagem caso a quantidade de utilitários aumente
12 // substancialmente.
13 //
14 // - Facilita-se a manutenção das implementações, pois estas ficam separadas,
15 // cada uma em seu ficheiro.
16 //
17 // Note que as implementações dos vários utilitários são fornecidas na forma de
18 // uma _biblioteca_, ou seja, na forma de um ficheiro que arquiva os ficheiros
19 // objecto correspondentes a cada ficheiro de implementação. Isso significa que
20 // a organização das implementações apenas num grande ficheiro ou, como aqui se
21 // sugere, em ficheiros separados para cada utilitário, _é totalmente
22 // transparente_ para o programador cliente da biblioteca.
23 //
24 // Note ainda que, em Linux, a biblioteca terá o nome `libarray_utils.a`.
25 
26 // Inclusão do cabeçalho correspondente a esta implementação
27 // ---------------------------------------------------------
28 //
29 // Antes de qualquer outra inclusão, incluímos o ficheiro de cabeçalho
30 // correspondente a este ficheiro de implementação. Isso ajuda a detectar
31 // inconsistências entre o ficheiro de implementação e o correspondente ficheiro
32 // de cabeçalho.
33 #include "array_utils.h"
34 
35 // Inclusão de ficheiros de cabeçalho necessários no código de implementação.
36 // Deve ser feita apenas _após_ se incluir o ficheiro de cabeçalho corresponde
37 // ao próprio ficheiro de implementação.
38 #include <stdio.h>
39 
40 // Definição da rotina (um utilitário) correspondente a este ficheiro de
41 // implementação. A definição das rotinas repete (porque o C o impõe) o seu
42 // cabeçalho, mas segue-o de um corpo onde se define o funcionamento ou
43 // implementação da rotina. Note que normalmente não se incluem comentários de
44 // documentação (e.g., [Doxygen](http://doxygen.org)) nos ficheiros de
45 // implementação.
46 void print(int number_of_items, int items[number_of_items])
47 {
48  for (int i = 0; i != number_of_items; i++)
49  printf("%d\n", items[i]);
50 }
