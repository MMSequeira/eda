14 // Coelhos, recursão e iteração
15 // ----------------------------
16 //
17 // Ofereceram ao Sr. Fibonacci um casal de coelhos, recém-nascidos, de um
18 // espécie curiosa: tornam-se férteis ao fim de um mês de vida, concebem logo
19 // que possível, têm uma gestação de exactamente um mês, cada ninhada consiste
20 // sempre num casalinho de coelhos. Nunca morrem. O Sr. Fibonacci preocupou-se:
21 // teria comida que chegue para os coelhos? Pôs-se a calcular a evolução do
22 // número de casais de coelhos ao longo do tempo:
23 //
24 // 0. No mês zero o Sr. Fibonacci vivia feliz, sem qualquer coelho. Mal
25 // calculava o seu destino...
26 //
27 // 1. No primeiro mês há um casal de coelhos inférteis.
28 //
29 // 2. No segundo mês há um casal de coelhos, já férteis e, claro está, com a
30 // coelha no início da gestação de mais um casalinho.
31 //
32 // 3. No terceiro mês há o casal de coelhos original, fértil, e já no início da
33 // gestação de mais um casalinho de coelhos. E há, claro está, um casal recém-
34 // nascido e, por isso, infértil. Ou seja, há dois casais de coelhos, um fértil
35 // e outro infértil.
36 //
37 // 4. No quarto mês há os dois casais do mês anterior, já férteis, adicionados
38 // de um casal recém-nascido, resultado do acasalamento do único casal fértil do
39 // mês anterior. Ou seja, há três casais de coelhos, dois férteis e um infértil.
40 //
41 // 5. No quinto mês há os três casais do mês anterior, já férteis, adicionados
42 // de dois casais recém-nascidos, resultado do acasalamento entre os dois casais
43 // férteis do mês anterior. Ou seja, há cinco casais de coelhos, três férteis e
44 // dois inférteis.
45 //
46 // O Sr. Fibonacci concluiu rapidamente que, a partir do terceiro mês,
47 //
48 // - o número de casais férteis num mês é sempre igual ao número total de casais
49 // no mês anterior e
50 //
51 // - o número total de casais num mês é sempre igual à soma do número total de
52 // casais existente no mês anterior com o número de casais recém-nascidos, que
53 // é igual ao número de casais férteis no mês anterior e que, por isso, é
54 // igual ao número total de casais existentes dois meses antes.
55 //
56 // Sendo ![F(n)](http://latex.codecogs.com/gif.latex?%5Cinline%20F_n) o número
57 // de casais de coelhos no mês
58 // ![n](http://latex.codecogs.com/gif.latex?%5Cinline%20n), então, desde que
59 // ![n>2](http://latex.codecogs.com/gif.latex?%5Cinline%20n%3E2), ![F(n)=F(n-2)+
60 // F(n-1)](http://latex.codecogs.com/gif.latex?%5Cinline%20F_n%3DF_%7Bn-2%7D&plus;F_%7Bn-1%7D). Quando
61 // ![n=1](http://latex.codecogs.com/gif.latex?%5Cinline%20n%3D1) ou
62 // ![n=2](http://latex.codecogs.com/gif.latex?%5Cinline%20n%3D2),
63 // ![F(n)=1](http://latex.codecogs.com/gif.latex?%5Cinline%20F_n%3D1). Ou seja,
64 // esta sucessão pode ser descrita de forma _recursiva_, ou seja, definindo os
65 // termos da sucessão em função de outros termos da mesma sucessão.
66 //
67 // O que pretendemos aqui é implementar uma função que receba o mês como
68 // argumento e devolva o número de casais existentes nesse mês. Como não
69 // fornecemos apenas uma implementação, mas várias, com diferentes eficiências,
70 // os nomes dessas funções não poderão ser simplesmente `fibonacci()`, incluindo
71 // no seu nome um sufixo que identifica o tipo de implementação usado. Pode
72 // argumentar-se que, sendo a eficiência parte da interface de um módulo, então
73 // é razoável usar esses sufixos. No entanto, seria preferível, se o nosso
74 // objectivo não fosse o estudo dos algoritmos em si, mas a criação de
75 // ferramentas para utilização futura ou para disponibilização a terceiros, usar
76 // sufixos que fossem reveladores da eficiência da função, e não da sua
77 // implementação.
78 //
79 // Antes, porém, de passar às implementações, é preciso dizer que o presente
80 // oferecido ao Sr. Fibonacci está realmente envenenado. Com efeito, a sequência
81 // de Fibonacci cresce de forma extremamente rápida. Assim, no mês 120, ou seja,
82 // dez anos depois da oferta do casal original, o número de casais a alimentar é
83 // exactamente 5&thinsp;358&thinsp;359&thinsp;254&thinsp;990&thinsp;966&thinsp;640&thinsp;871&thinsp;840.
84 // Este valor é demasiado grande para os `int` nas nossas máquinas habituais,
85 // que têm 32 _bits_, ou mesmo para os _long_, com 64 _bits_ nessas mesmas
86 // máquinas. Por outro lado, a precisão limitada dos tipos de virgula flutuante
87 // pode pôr problemas complicados ao cálculo dos termos da sequência (já
88 // tínhamos dito que dos tipos de vírgula flutuante se foge como diabo da
89 // cruz?). O código abaixo recorre ao tipo `long`, numa tentativa de evitar que
90 // se atinjam muito cedo os limites dos inteiros. É uma ná solução, no entanto.
91 // Uma boa solução passaria pela utilização de um tipo capaz de representar
92 // números inteiros de dimensão arbitrária (desde que haja memória para o
93 // guardar, bem entendido). A implementação de um tal tipo é um excelente
94 // exercício de algoritmos e estruturas de dados, mas não o resolveremos aqui.

179 long iterative_fibonacci(int n)
180 {
181  assert(n >= 0);
182 
183  if (n == 0)
184  return 0L;
185  if (n == 1)
186  return 1L;
187 
188  long previous_term = 1L;
189  long current_term = 1L;
190  for(int i = 2; i != n; i++) {
191  current_term += previous_term;
192  previous_term = current_term - previous_term;
193  }
194 
195  return current_term;
196 }
