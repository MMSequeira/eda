4 // O que são os _arrays_
5 // ---------------------
6 //
7 // Os _arrays_ são uma das formas de agregar dados em C. Nos _arrays_ os dados,
8 // a que chamamos «itens», são todos do mesmo tipo. Estes dados estão
9 // organizados numa sequência com um comprimento fixo. Por exemplo,
10 //
11 // ```C
12 // int array[4];
13 // ```
14 // define um novo _array_ com exactamente quatro inteiros.
15 //
16 // Inicalização de _arrays_
17 // ------------------------
18 //
19 // ### Inicialização por omissão
20 //
21 // Tal como acontece no caso de variáveis dos tipos básicos da linguagem, por
22 // omissão o C _não_ inicializa os itens de um _array_ (desde que seja um
23 // _array_ automático, pois se for estático todos os itens são inicializados com
24 // um valor nulo). Ou seja, estes começam a sua vida com valores que, do ponto
25 // de vista do programador, são puro lixo, mesmo que provavelmente estes valores
26 // consistam simplesmente numa interpretação do padrão de _bits_ que se encontra
27 // na memória alocada para esses itens ou variáveis.
28 //
29 // ### Inicialização explícita
30 //
31 // Os _arrays_ podem ser inicializados explicitamente através de uma lista
32 // de inicializadores:
33 //
34 // ```C
35 // int array[4] = {0, 1, 2};
36 // ```
37 //
38 // Neste caso omitimos o valor inicial do quarto item do _array_. Quando isto
39 // acontece, o compilador do C encarrega-se de gerar código máquina que
40 // inicializa os itens em falta com valores nulos. Podemos, por isso, usar um
41 // truque simpático para inicializar todos os itens dos _arrays_ do C com o
42 // valor zero (no entanto, a norma que especifica o C11 não é clara quanto à
43 // validade de usar uma lista de inicialização vazia...):
44 //
45 // ```C
46 // int array[4] = {};
47 // ```
48 //
49 // Quando se fornece uma lista de inicialização, é possível omitir a
50 // dimensão dos _arrays_. Nesse caso, o _array_ terá como dimensão o número
51 // de valores colocado entre chavetas. Por exemplo, o código
52 //
53 // ```C
54 // int array[] = {0, 1, 2};
55 // ```
56 //
57 // define um _array_ com exactamente três itens.

75 // esteja também disponível uma biblioteca ou um ficheiro objecto com o
76 // resultado da compilação do ou dos ficheiros de implementação correspondentes.

81 // Definição de uma função
82 // =======================
83 //
84 // Definimos uma função que devolve a média dos itens da matriz passada como
85 // argumento. Uma definição de uma rotina consiste num _cabeçalho_, onde se
86 // caracteriza a interface da rotina, e num _corpo_, onde se coloca a
87 // _implementação_ da rotina. Neste caso temos uma _definição_, e não uma
88 // simples _declaração_, pois incluímos o corpo da função. Todas as definições
89 // são também declarações, mas nem todas as declarações são definições. Uma
90 // declaração simples, sem indicação da implementação, inclui apenas o cabeçalho
91 // da rotina seguido de um ponto e vírgula. O ficheiro
92 // [`array_utils.h`](array_utils.html) contém apenas a declaração da função.
93 
94 // ### Documentação e cabeçalho da função
95 //
96 // O _cabeçalho_ de uma rotina declara a sua existência e especifica a sua forma
97 // de utilização, mas não especifica totalmente a _interface_ da função. A
98 // interface completa de uma rotina inclui informação mais pormenorizada acerca
99 // dos objectivos da rotina, acerca do significado dos parâmetros, etc., mas,
100 // sobretudo, estabelece um contrato claro entre o _programador produtor_ da
101 // rotina, i.e., o ou os programadores que a implementam e mantêm, e o
102 // _programador consumidor ou cliente_ da rotina, i.e., o ou os programadores
103 // que a usam no seu código. O C não suporta directamente a especificação da
104 // interface completa da rotina. Por isso, essa especificação é feita recorrendo
105 // a um _comentário de documentação_.
106 
107 // #### Documentação
108 //
109 // O cabeçalho é precedido de um _comentário de documentação_ que indica
110 // claramente qual o contrato desta função. Usamos aqui o formato de
111 // documentação [Doxygen](doxygen.org).
122 //
123 // #### Cabeçalho
124 //
125 // Este cabeçalho inclui a seguinte informação:
126 //
127 // - Nome da função &ndash; O nome da função deve ser tão explícito e claro
128 // quanto possível. Pense numa possível utilização da função para chegar a um
129 // bom nome. Por exemplo, suponha-se que existe um _array_ chamado
130 // `student_grades` com a lista das notas de um dado aluno e se pretende saber
131 // se a sua média ultrapassa os 15 valores. O código a escrever seria o que se
132 // segue, que é legível quase como se se tratasse de inglês:
133 //
134 // ```C
135 // if (average_of(student_grades) > 15.0)
136 // ...
137 // ```
138 //
139 // - Tipo do valor devolvido &ndash; !!!!!!!!!!!!!!!!!!!
140 double average_of(int number_of_items, int items[number_of_items])
141 //double average_of(int* items)
142 {
143  double sum = 0.0;
144 
145  for (int i = 0; i != number_of_items; i++)
146  sum += items[i];
147 
148  return sum / 4;
149 }
