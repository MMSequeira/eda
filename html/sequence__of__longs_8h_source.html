59 // ### Declaração da `struct` que representa as sucessões de `long`
60 //
61 // Note que não é uma definição da estrutura, mas sim uma sua mera declaração.
62 // Sem acesso à definição da estrutura, o código cliente deste módulo físico não
63 // pode declarar directamente variáveis do tipo `struct sequence_of_longs`
64 // (e.g., na pilha), nem tão pouco pode reservar directamente variáveis
65 // dinâmicas desse tipo (no monte), usando `malloc()` e recorrendo ao operador
66 // `sizeof`: sem acesso à definição da estrutura, o compilador não pode
67 // adivinhar quanto espaço deve reservar para estas variáveis, seja na pilha
68 // (_stack_), seja no monte (_heap_).
69 //
70 // A definição da estrutura neste local, em vez da declaração existente,
71 // eliminaria estas restrições. No entanto, levaria a uma pior separação entre
72 // interface e implementação, pois alterações na forma de representação do TAD
73 // levariam quase inevitavelmente a alterações na definição da estrutura e, por
74 // isso, a alterações neste ficheiro de interface e, consequentemente, à
75 // necessidade de recompilar todo o código cliente deste módulo físico, ou seja,
76 // todo o código que inclui este ficheiro de interface.
77 //
78 struct sequence_of_longs;
