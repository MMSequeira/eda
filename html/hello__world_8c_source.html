1 // `hello_world.c` &ndash; Estrutura de um programa em C
2 // =============================
3 //
4 // Este programa ilustra a estrutura básica de um programa escrito na linguagem
5 // C. O objectivo deste programa é imprimir no ecrã o factorial de um número
6 // inteiro introduzido pelo utilizador do programa.
7 //
8 // Avisos!
9 // -------
10 //
11 // De modo a mantê-lo curto, optámos por omitir deste programa todo o código de
12 // verificação de erros do utilizador. _Todos os programas têm de saber lidar
13 // com erros dos seus utilizadores!_ Não o fazer é um erro _grave_. Este
14 // programa está, assim, _gravemente errado_. Não diga que não o avisámos. :-)
15 //
16 // Ficheiro e nome
17 // ---------------
18 //
19 // Este programa consiste numa única _unidade de compilação_. Ou seja,
20 // corresponde a um único ficheiro destinado a ser _compilado_. Os ficheiros em
21 // linguagem C que se destinam a ser compilados possuem convencionalmente a
22 // extensão `.c`. O nome do ficheiro é, assim, `hello_world.c`.
23 //
24 // Este ficheiro não é suficiente para _construir_ o ficheiro executável, pois
25 // recorre a ferramentas da _biblioteca padrão_ do C. Em particular, recorre a
26 // rotinas de leitura e escrita de dados e recorre à _macro_ `EXIT_SUCCESS`.
27 //
28 // A construção do executável
29 // --------------------------
30 //
31 // Os passos usuais na construção de um ficheiro executável a partir de um
32 // programa em linguagem C são os seguintes:
33 //
34 // 1. Pré-processar cada um dos ficheiros `.c` - O pré-processador é um programa
35 // que lê um ficheiro contendo código C e directivas de pré-processamento e
36 // interpreta estas directivas, produzindo um ficheiro contendo linguagem C
37 // apenas (bem como algumas directivas de pré-processamento indicando a
38 // proveniência das linhas de código). A este ficheiro contendo apenas
39 // linguagem C (ou quase), damos o nome de _unidade de compilação_.
40 // Usualmente as unidades de compilação não têm senão uma existência efémera,
41 // podendo mesmo não chegar a existir enquanto ficheiros no sistema de
42 // ficheiros do computador.
43 //
44 // 2. Compilar cada um dos ficheiros resultantes do pré-processamento, ou seja,
45 // compilar cada uma das unidades de compilação. A compilação é o processo de
46 // tradução da linguagem C para a linguagem máquina do computador em causa. O
47 // resultado da compilação de uma unidade de compilação é um _ficheiro
48 // objecto_. Os ficheiros objecto têm, em Linux, a extensão `.o`. Em Windows
49 // é típico a extensão ser `.obj`. Os ficheiros objecto não são ainda
50 // executáveis, pois tipicamente cada ficheiro objecto contêm código máquina
51 // que recorre a ferramentas externas a esse ficheiro objecto.
52 //
53 // 3. Depois de gerados os ficheiros objecto, estes têm de ser fundidos num
54 // único ficheiro executável. A fusão resolve as dependências entre os vários
55 // ficheiro objecto e entre estes e colecções de ficheiros objecto arquivados
56 // em _bibliotecas_. As bibliotecas, em Linux, têm a extensão `.a` (de
57 // _archive_), quando estáticas, ou `.so`, quando dinâmicas. Em Windows as
58 // extensões correspondentes são `.lib` e `.dll`. O resultado de uma fusão
59 // com sucesso é um ficheiro executável. Em Linux os ficheiros executáveis
60 // podem ser qualquer extensão. Em Windows, a extensão de um ficheiro
61 // executável é `.exe`.
62

64 // ----------------------------------

66 // É usual os ficheiros `.c`, ou seja, os ficheiros que são pré-processados, se
67 // É iniciem com !!!!!!!!!!!!!!

72 int factorial(const int n)
73 {
74  if (n == 0)
75  return 1;
76  return n * factorial(n - 1);
77 }
