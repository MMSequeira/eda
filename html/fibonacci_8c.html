Detailed Description

Illustrates possible implementations of the calculation of terms of the Fibonacci sequence.

Iterative and recursive implementations of the calculation of terms of the Fibonacci sequence. These implementations are used to illustrate iteration vs. recursion, and also to show how a naïve implementation of recursion may lead to serious problems and how lookup tables can be used to speed up calculations.

Experiments a given Fibonacci sequence term calculation function for a sequence of terms and reports the extimated execution times to stdout.

Parameters
titleThe title of the experiment.
fibonacciPointer to the function to experiment with.
last_term_to_testThe function will be experimented with terms from 0 up to this number.
Precondition
title ≠ null
fibonacci ≠ null
last_term_to_test ≥ 0
Postcondition
The results of the experiment are writen in stdout.

This procedure performs experiments with the provided Fibonacci function fibonacci(), reporting the estimated execution times for terms 0 up to last_term_to_test. In order to overcome clock resolution issues, each experiment is repeated until a total of at least 0.1 seconds is reached. For all but the first term, the relative increases in execution time are also reported.

Bug:
The minimum accumulated time should not be hardcoded as 0.1 seconds. It should be a parameter of the procedure instead.

long iterative_fibonacci ( int  n)

Returns the nth term of the Fiboncci sequence.

Parameters
nThe number of the term to return (first valid value is 0).
Returns
The value of the nth term \(F_{\mathtt{n}}\) of the Fibonacci sequence.
Precondition
n ≥ 0
Postcondition
result = \(F_{\mathtt{n}}\)

Returns the nth term of the Fibonacci sequence, i.e., \(F_{\mathtt{n}}\). It is assumed the sequence \(F_n\) starts at \(n=0\), with value 0, followed by value 1, at \(n=1\). That is, the sequence is defined by

\[ F_n = \left\{\begin{array}{ll} 0 & \text{if } n=0, \\ 1 & \text{if } n=1\text{, and} \\ F_{n-2} + F_{n-1} & \text{if } n>1. \end{array}\right. \]

It can be shown that

\[ F_n = \frac{\phi^n-\psi^n}{\sqrt{5}}, \]

where \(\phi=\frac{1+\sqrt{5}}{2}\) and \(\psi=\frac{1-\sqrt{5}}{2}\), and also that, even more simple,

\[ F_n = \left[\frac{\phi^n}{\sqrt{5}}\right], \]

where \([\cdot]\) is the nearest integer function.

The time taken by the function grows linearly with n. More precisely, the number of aditions and subtractions performed is 0 if \(n=0\) or \(n=1\), and \(2(n-2)\) if \(n>1\).

See tail_recursive_fibonacci() for a fast, tail recursive implementation of the Fibonacci sequence with approximately the same efficiency as this implementation.

long recursive_fibonacci ( int  n)

The value of the nth term \(F_{\mathtt{n}}\) of the Fibonacci sequence.
Precondition
n ≥ 0
Postcondition
result = \(F_{\mathtt{n}}\)

Returns the nth term of the Fibonacci sequence, i.e., \(F_{\mathtt{n}}\). See stupidly_recursive_fibonacci() for further information.

This function uses a static array of MAXIMUM_TERM_FITTING_A_LONG longs to store calculated values of the terms of the sequence. After a call to the function with argument \(n\), all future calls to the function with any argument between 0 and \(n\) will execute in constant time. When this doesn't apply, the function will execute in linear time, i.e., \(O(n)\). More precisely, while calculating \(F_n\) by calling recursive_fibonacci( \(n\)),

  • the number \(N(n)\) of (recursive) executions of this function is 1 if \(n=0\) and is \(2n-1\) if \(n>0\);
  • the number \(S(n)\) of aditions performed is 0 if \(n=0\) and is \(n-1\) if \(n>0\);
  • the number \(T(n)\) of array item assignments performed is 1 if \(n=0\) or \(n=1\), and is \(n+1\) if \(n>1\);
  • the number \(R(n)\) of array item reads performed is 0 if \(n=0\) or \(n=1\), and is \(n-2\) if \(n>1\).

These numbers are exact if no other calls to the function have been performed previously, otherwise the values will be either smaller or equal to these. If other calls with an argument larger or equal to \(n\) were performed previously, then \(N(n)=1\), \(S(n)=0\), \(T(n)=0\), and \(R(n)=1\).

Postcondition
result = \(F_{\mathtt{n}}\)

Returns the nth term of the Fibonacci sequence, i.e., \(F_{\mathtt{n}}\). See stupidly_recursive_fibonacci() and recursive_fibonacci() for further information.

This function uses the ADT struct sequence_of_longs store calculated values of the terms of the sequence. After a call to the function with argument \(n\), all future calls to the function with any argument between 0 and \(n\) will execute in constant time. When this doesn't apply, the function will execute in linear time, i.e., \(O(n)\). More precisely, while calculating \(F_n\) by calling recursive_fibonacci_using_ADT( \(n\)),

  • the number \(N(n)\) of (recursive) executions of this function is 1 if \(n=0\) and is \(2n-1\) if \(n>0\);
  • the number \(S(n)\) of aditions performed is 0 if \(n=0\) and is \(n-1\) if \(n>0\);
  • the number \(T(n)\) of additions to the memory of terms is 1 if \(n=0\) or \(n=1\), and is \(n+1\) if \(n>1\);
  • the number \(R(n)\) of accesses to the memory of terms is 0 if \(n=0\) or \(n=1\), and is \(n-2\) if \(n>1\).

These numbers are exact if no other calls to the function have been performed previously, otherwise the values will be either smaller or equal to these. If other calls with an argument larger or equal to \(n\) were performed previously, then \(N(n)=1\), \(S(n)=0\), \(T(n)=0\), and \(R(n)=1\).

Additions of calculated terms to the term memory are less efficient when the ADT struct sequence_of_longs is used than when a fixed size array is used. However:

  • The ADT guarantees that the amortized number of copies of terms perfomed while adding a new term is always less than 2, approaching 1 as the number of terms in the sequence grows to infinity.
  • An ADT such as the one used would be necessary if big integers were used for the result of this function.

See iterative_fibonacci() and tail_recursive_fibonacci() for fast implementations of the Fibonacci sequence that do not require static storage and that execute in linear time.
